## 3.1 Strona "Polityka prywatności / RODO"

### Cel
Profesjonalna strona informacyjna dostępna pod `/polityka-prywatnosci`, wymagana przez RODO dla firm B2B. Powinna być czytelna, rzeczowa, z podziałem na sekcje.

### Nowy plik: `src/pages/PrivacyPage.jsx`
Struktura strony:
```
<section> - nagłówek "Polityka prywatności"
  <div> - breadcrumb lub link powrotu na stronę główną
<section> - 1. Administrator danych (dane firmy Partner-Net)
<section> - 2. Jakie dane zbieramy
  - Dane z formularza kontaktowego (imię, e-mail, telefon, NIP, wiadomość)
  - Dane z plików cookies (analityczne, marketingowe)
<section> - 3. Cel przetwarzania danych
  - Obsługa zapytań ofertowych
  - Analityka ruchu na stronie
<section> - 4. Podstawa prawna
  - Zgoda użytkownika (art. 6 ust. 1 lit. a RODO)
  - Prawnie uzasadniony interes (art. 6 ust. 1 lit. f RODO)
<section> - 5. Pliki cookies
  - Link do otwarcia panelu ustawień cookies (dispatch 'openCookieSettings')
  - Opis typów: niezbędne, analityczne, marketingowe
<section> - 6. Prawa użytkownika
  - Dostęp, sprostowanie, usunięcie, ograniczenie, przenoszenie, sprzeciw
  - Prawo do cofnięcia zgody
  - Prawo do skargi do UODO
<section> - 7. Okres przechowywania
<section> - 8. Kontakt w sprawie danych
  - Email, telefon
<section> - 9. Aktualizacja polityki
```

Design: biała karta z cieniem, maks szerokość `max-w-3xl`, każdy nagłówek sekcji jako `h2` z numeracją, treść `text-sm leading-relaxed text-slate-700`. Zgodny ze stylem reszty strony.

## 3.4 CookieConsent refactor

### Cel
Obecny CookieConsent (179 linii) działa dobrze, ale ma kilka braków:
- Brak linku do polityki prywatności
- Customowy Switch zamiast natywnego `<input type="checkbox" role="switch">`
- Overlay na settings nie do końca estetyczny
- Opisy kategorii cookies są bardzo ogólne (po 1 zdaniu)

### Modyfikacja `src/components/ui/CookieConsent.jsx`

#### Zmiany strukturalne:
1. **Zastąpić customowy Switch** natywnym `<input type="checkbox" role="switch">` z lepszym stylowaniem CSS (bezpieczniej dla dostępności, screen readerów)
2. **Dodać link do polityki prywatności** w widoku głównym i w ustawieniach:
   ```jsx
   import { Link } from 'react-router-dom'
   // w widoku głównym pod tekstem:
   <Link to="/polityka-prywatnosci" className="text-xs text-[#00aeff] hover:underline mt-1 inline-block">
     Więcej w polityce prywatności →
   </Link>
   ```
3. **Rozszerzyć opisy kategorii** cookies o konkretne przykłady:
   - Analityczne: "Google Analytics (zanonimizowane IP), SWR cache. Pozwalają mierzyć ruch i optymalizować wydajność."
   - Marketingowe: "Meta Pixel, LinkedIn Insight Tag. Umożliwiają wyświetlanie dopasowanych reklam."
4. **Poprawić design overlay** dla ustawień:
   - Obecnie overlay jest przezroczysty + blur - zmienić na `bg-black/40`
   - Dodać płynniejsze przejście między widokiem głównym a settings (już jest Framer Motion, wystarczy drobne poprawki)
5. **Dodać przycisk "Tylko niezbędne"** w głównym widoku (oprócz "Akceptuj wszystkie" i "Ustawienia"):
   ```jsx
   <Button onClick={handleAcceptNecessary} variant="ghost" className="...">
     Tylko niezbędne
   </Button>
   ```
6. **Lepsze odstępy na mobile** - obecnie przyciski są ciasno na małych ekranach.

#### Nowa funkcja `handleAcceptNecessary`:
```js
const handleAcceptNecessary = () => {
  const prefs = { necessary: true, analytics: false, marketing: false }
  setPreferences(prefs)
  localStorage.setItem('sn_cookie_consent', JSON.stringify(prefs))
  setIsOpen(false)
}
```

#### Uproszczenie kodu:
Obecny kod jest w porządku. Główne zmiany to:
- Import `Link` z react-router-dom (nowy)
- Dodanie `handleAcceptNecessary` (nowa funkcja ~5 linii)
- Zamiana Switch na input checkbox (~10 linii zmiany)
- Rozszerzenie opisów (~zmiana tekstów)
- Dodanie linku do polityki (~3 linie JSX)
- Lekkie poprawki stylowania (~zmiana klas Tailwind)

## 3.5 Favicon i PWA meta

### Cel
Kompletny zestaw ikon i meta tagów dla:
- Przeglądarek (favicon)
- iOS (apple-touch-icon)
- Android (manifest.json)
- Kolor paska przeglądarki (theme-color)

### Nowy plik: `public/manifest.json`
```json
{
  "name": "Partner Numery Seryjne",
  "short_name": "Partner SN",
  "description": "Moduł B2B do zarządzania numerami seryjnymi dla Subiekt nexo PRO",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#f8fafc",
  "theme_color": "#00aeff",
  "icons": [
    {
      "src": "/favicon.svg",
      "sizes": "48x46",
      "type": "image/svg+xml"
    }
  ],
  "lang": "pl"
}
```

### Modyfikacja `index.html`

Dodać w `<head>`:

```html
<!-- PWA -->
<link rel="manifest" href="/manifest.json" />
<meta name="theme-color" content="#00aeff" />
<meta name="apple-mobile-web-app-capable" content="yes" />
<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent" />
<meta name="apple-mobile-web-app-title" content="Partner SN" />

<!-- Favicons (dodatkowe formaty dla kompatybilności) -->
<link rel="icon" type="image/svg+xml" href="/favicon.svg" />
<link rel="apple-touch-icon" href="/favicon.svg" />

<!-- Optymalizacja: preload fontów -->
<link rel="preload" href="https://fonts.googleapis.com/css2?family=Inter:wght@300;400;600;700&display=swap" as="style" onload="this.onload=null;this.rel='stylesheet'" />
<noscript><link href="https://fonts.googleapis.com/css2?family=Inter:wght@300;400;600;700&display=swap" rel="stylesheet" /></noscript>

<!-- Meta dodatkowe -->
<meta name="robots" content="index, follow" />
<link rel="canonical" href="https://twojadomena.pl" />
```

Uwagi:
- Obecnie favicon.svg ma rozmiar 48x46 - to niestandardowe. SVG i tak będzie skalowany, więc to nie problem dla przeglądarek.
- `apple-touch-icon` standardowo oczekuje PNG 180x180. Użycie SVG może nie działać na starszych iOS. Opcjonalnie: dodać `/favicon-180.png` jako fallback. Ale na razie SVG jest ok.
- `manifest.json` jest już linkowany przez `<link rel="manifest">`.
- Font preload z `onload` trickiem to optymalizacja - font ładuje się asynchronicznie bez blokowania renderowania.

### Istniejące meta tagi do zachowania:
Obecny `index.html` ma już: `charset`, `viewport`, `og:type`, `og:title`, `og:description`, `description`, `title`. Wszystkie zostają. Dodajemy tylko PWA/icon meta i poprawiamy font loading.
